CollectorHub Update 34 — JOIN ALL + TITLE ONLY

Что исправлено:
1. Полностью убран подход "сохранили часть, остальные пропустили".
   Теперь программа обрабатывает ВСЕ найденные группы по всем ключевым словам.

2. Фильтрация только по названию группы.
   Используются только h1/title страницы группы и название из карточки поиска.
   Описания, публикации, комментарии и раздел "О группе" не читаются и не используются.

3. Криво прочитанная карточка больше не убивает результат.
   Если в карточке поиска Facebook не дал нормальное название, программа открывает группу
   и читает только заголовок группы, а уже потом решает: подходит название или нет.

4. Сохранение только после вступления.
   Группа добавляется в список только если:
   - уже были участником; или
   - кнопка "Присоединиться/Join/Dołącz" нажата;
   - после ожидания появился подтверждённый статус участника.

5. Если одна группа не вступила — процесс не падает.
   Она пишется в журнал как "НЕ сохраняю", а программа идёт к следующей группе.

6. В поле name/title пишется человеческое название группы, не ID.
   ID и ссылка остаются только техническими данными.

7. Дубли не создаются.
   Существующая группа обновляется по URL/group_id.

Важно:
- Если Facebook показывает вопросы для вступления или только отправляет заявку pending,
  группа НЕ сохраняется, потому что вступление ещё не подтверждено.
- Если кнопка visible в браузере, программа пробует нажимать её несколькими селекторами
  и не останавливает весь поиск из-за одной неудачной группы.
